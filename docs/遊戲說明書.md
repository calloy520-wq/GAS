# 《海與劍之歌》遊戲設計說明書

> 給後續開發者／AI 的完整功能地圖。內容以程式碼實作為準（Data.gs / Engine.gs / Store.gs / Code.gs / Index.html），
> 若日後改動，請同步更新本檔。最後校對版本：見 git log。

---

## 0. 一句話定位

免費、可多人的**網頁 RPG**，跑在 Google Apps Script 上。三條主軸交織：

- **⚔️ 劍（地下城）**：D&D 式自動探索 + 回合制戰鬥（陸地起手區）。
- **🌊 海（大航海）**：買賣跑商、海戰掠奪、攻城收稅（第 5 層領船後解鎖）。
- **👑 爭霸**：威名／爵位總目標、艦隊帝國、副手情義（後期經營）。

風格參考：**傭兵團 × 大航海時代 × 騎馬與砍殺**。玩家是傭兵團團長／船長。

## 1. 技術架構

### 1.1 執行環境
- **Google Apps Script（V8）** 網頁應用程式。時區 Asia/Taipei。
- `doGet()` → 回傳 `Index.html`（單一 SPA）。
- 前端唯一後端入口：`google.script.run.api(action, payloadJson)`。
- 資料庫：**綁定的 Google Sheet**，分頁 `Players`（見 Store.gs）。
- OAuth scope 僅需 `spreadsheets`。網頁存取 `ANYONE_ANONYMOUS`、`executeAs: USER_DEPLOYING`。

### 1.2 檔案分工
| 檔案 | 行數量級 | 職責 |
|---|---|---|
| `Data.gs` | ~400 | **規則常數與資料表**：職業/種族/稀有度/裝備/怪物/貨品/港口/船艦/威名/世界事件。純資料 + 少量純函式。 |
| `Engine.gs` | ~380 | **骰子與戰鬥引擎**：建角、升級、屬性、技能檢定、`runDungeon()` 地城 DM、`resolveCombat()` 回合戰。 |
| `Store.gs` | ~200 | **資料層**：Sheet 讀寫、快取、`ensureShape_()` 補洞、`sanitize_()` 夾值、名冊 `listPlayers()`。 |
| `Code.gs` | ~860 | **路由與所有 API 端點**：`api()`→`route_()`→`apiXxx_()`。海事等級、海戰、貿易、艦隊、領地、職務等伺服器邏輯。 |
| `Index.html` | ~1980（長行） | **整個前端 SPA**：CSS + 所有畫面渲染 + 離線 mock（伺服器邏輯的鏡像）。 |
| `appsscript.json` | — | GAS 專案設定。 |
| `.clasp.json` / `.claspignore` | — | clasp 部署設定；**只推送 `*.gs` / `*.html` / `appsscript.json`**（`.md` 不上傳）。 |
| `.github/workflows/*.yml` | — | push 到 `main` → clasp push + clasp deploy（更新既有部署，`/exec` 網址不變）。 |

### 1.3 資料流
```
前端 srv(action,payload)
  └─ 線上：google.script.run.api(action, JSON) → route_ → apiXxx_ → {ok,data} JSON 字串
  └─ 離線：mock(action,p) 讀寫 localStorage（無網路時的鏡像，含 250ms 假延遲）
存檔寫入：saveP(cb) 樂觀更新（cb 先改畫面）→ 300ms 去抖 → doSaveNow → srv('save')
  重讀伺服器前一定先 flushSave()，避免舊資料覆蓋。
```

### 1.4 伺服器權威（authoritative）
所有隨機、戰鬥、經濟結算都在**後端**算完再回傳結果。前端只負責畫面與動畫播放。
- 地城戰：`runDungeon` 產生完整戰報，內含結構化事件 `floors[].encounters[].ev`（給前端播動畫）。
- 海戰：`resolveNaval_` 產生 `nev`（naval events）+ `snap`（雙方船圖/血量上限）給動畫。

### 1.5 離線 mock（重要）
`Index.html` 內有一整套 `*C` / `*_C` 後綴的函式與常數（如 `tradePriceC`、`MARKETS_C`、`resolveNavalC`、`GARRISONS_C`），
是伺服器邏輯的**前端鏡像**，讓無網路（例如本機開檔）也能玩。
> ⚠️ **改任何伺服器規則／資料，務必同步更新 Index.html 的鏡像**，否則線上與離線行為不一致。

## 2. 存檔資料模型（player 物件）

存進 Sheet `Players.json` 欄的核心結構。`ensureShape_()`（Store.gs）保證舊檔補上新欄位。

```js
{
  nick, created, updated, ver,
  gold, deepest,              // 金幣、地城最深層（晉升/威名/敵強度都看它）
  heroId,                     // 主角角色 id
  roster: [char, ...],        // 全體角色（含主角），上限 CFG.ROSTER_MAX=10
  bag: [gearId, ...],         // 未裝備的裝備 id
  team: { battle:[id,...], support:[id,...] },  // 出戰 4 / 後勤 2
  tavern: [cand, ...],        // 酒館當前候選（後端抽定資質）
  quest: {id,type,target,prog,name,reward} | null,  // 公會委託
  questsDone,                 // 完成委託數（算威名）
  bonds: { "idA|idB": pts },  // 羈絆好感（同隊出戰累積，火紋式）
  uniques: { 5:true, ... },   // 已首殺的頭目樓層（神器只給一次）

  // ── 海洋（第 5 層領船後才有 ship）──
  ship: { name, hullMax, hull, cannon, cargoBonus, crew, speed, gunTier, tier } | undefined,
  port,                       // 目前停泊港 id（預設 'merc'）
  cargo: { goodId: qty },     // 船艙貨物
  clues,                      // 秘寶海圖殘片（集 5 挖寶）
  invest,                     // 商會投資額（每趟航程領分紅）
  fleet: [fleetShip, ...],    // 艦隊船（買的或鹵獲的），上限 fleetCap_()
  pendingPrize,               // 接舷俘虜到、待玩家決定編入/拆解的船
  holdings: { portId:{lv,since,lastAt} },  // 領地（攻城取得，收被動稅）
  posts: { navigator:id, gunner:id, quartermaster:id, lookout:id },  // 船上職務
  raided: { "day_npcId":true },            // 今日已掠奪的 NPC（防重複）
  scout: { key, gun } | null,              // 偵查到的奇襲先手（一次性）
  rep: { guild, navy, pirate, league },    // 四大勢力好感度（-1000~+1000）
  pendingCaptain: {nm,fac,ransom} | null,  // 接舷俘虜的敵船船長，待贖金/賣/放
  pendingRecruit: {char,ransom} | null,    // 招降的敗方船員，待收編/贖金

  // ── 帳號級養成 ──
  nav: {lv,xp},  com: {lv,xp},             // 航海術 / 商業（像騎砍團長技能）
  mate,                                    // 副手角色 id
  affinity: { id: pts },                   // 副手情義點數（affLevel 0~5）
}
```

角色 `char` 物件（Engine.gs `makeChar`）：
```js
{ id, name, job, race, support, portrait, seed,
  level, xp, base:{str,dex,con,int,wis,cha},
  equip:{weapon,armor,trinket}, hp, maxhp,
  rarity,                 // 招募稀有度
  traits:[key,...] }      // 探索得到的怪癖/稱號（改屬性）
```

## 4. 陸地系統（劍）

### 4.1 創角與角色
- **六大屬性** str/dex/con/int/wis/cha（D&D）。`mod = ⌊(score-10)/2⌋`。熟練 `2+⌊(lv-1)/4⌋`。
- **戰鬥職業**（`COMBAT_CLASSES`，填戰鬥位、下場打）：戰士/聖騎士/盜賊/遊俠/法師/牧師。各有生命骰 hd、主攻屬性 atk、成長 grow、招牌技 skill。
- **後勤職業**（`SUPPORT_CLASSES`，填後勤位、給全場增益）：吟遊詩人(atk+2)/商人(金幣+40%)/學者(經驗+35%)/醫者(每層回12%)/占卜師(暴擊+10%)。
- **種族**（`RACES`，創角一次性屬性加值）：人類/精靈/黑暗精靈/獸耳/龍族/魔族/天使/吸血鬼。
- **稀有度**（`RARITY`，招募資質）：常見→精良→稀有→史詩→傳說（權重 50/27/15/7/1；越稀有屬性越高、費用越貴；傳說需最深層≥12 才會出）。
- **技能專長**（`SKILLS`）：由職業＋種族決定（`charSkills`），探索時做 d20 檢定（察覺/隱匿/運動/醫療/求生/巧手/交涉）。
- **怪癖/稱號**（`TRAITS`）：探索事件留下的好/壞特質，改屬性（如屠龍者 str+1、舊傷 con-1）。上限 4。

### 4.2 裝備（Data.gs）
- 武器 `WEAPONS`（傷害骰+命中）、防具 `ARMORS`（AC）、飾品 `TRINKETS`（屬性+2）。分 tier 1~4。
- **獨門神器 `UNIQUE_GEAR`**：頭目樓層（5/10/15/20/25）**首殺限定**，飾品欄多屬性。每個只給一次（`player.uniques`）。

### 4.3 地下城（Engine.gs `runDungeon`）— 核心玩法
- **出發點＝checkpoint 概念**（不是任選中間層）：探索過的樓層留了繩索/記號，預設「🪜 垂降到最深第 N 層」續探（免重爬前段）；另可選「🏛️ 從入口第 1 層」穩紮練級／回頭清淺層。只選「這趟潛到第 __ 層」目標，不再手動填任意起始層。內部仍是 start/target（checkpoint→start=maxStart、入口→start=1）。
- **UX（重複掃蕩／編隊）**：地城面板頂端「編隊一覽」（`partyStripHTML`：前排戰鬥＋後勤頭像＋「調整編隊」捷徑），下城前看得到誰上場；目標層記在 `P.lastDive` 下次自動帶入（出發點預設回 checkpoint）；戰報有「🔁 再掃一次（第 s→t 層）」（`doDive` 共用，保留原設定重複掃蕩），受傷時多一顆「🏨 療傷再戰」（費用同旅館 `15+深潛×2`、破產免費）。
- 每層：探索技能檢定（察覺找財寶/陷阱/隱匿偷襲先手）→ 遭遇戰 `resolveCombat` → 結算金幣/經驗/掉落。
- **打贏一層＝紮營回滿血**（存活的戰鬥位 `h.hp=h.maxhp`）：可連續深潛、不必一層一趟跑回城補血，破除來回補血的疲勞；死亡只發生在「單層戰鬥中被打死」，非血量慢慢流乾。**醫者（後勤 `buff.heal`）改為每層就地救回一名陣亡夥伴**（回 `maxhp*min(1,heal*4)` 血），full-heal 下仍有明確價值。頭目層（每 5 層）是主要難關，中途全滅照樣失去該趟戰利品。
- **每 5 層是頭目層**（`BOSSES`：哥布林王/巨魔領主/骨龍/巫妖）。頭目必掉裝備、有機率給正面稱號。
- **回合戰**：我方行動（蓄力 3 回合放招牌技，否則普攻）→ 敵方行動。d20 命中比 AC。
- **蓄力技**（`heroSkill`）：aoe火球/heal聖光復活/multi三連射/double猛擊/smite神聖斬/sneak偷襲。
- **全滅懲罰**：金幣與戰利品被救援費拿走一半，倒下夥伴回旅館付費救治。
- **羈絆**（火紋式）：同隊出戰每對 +2（上限60），等級 C/B/A/S 給同隊加攻。
- **主線晉升**（`RANKS`）：靠最深層自動升「見習→銅→銀→金→白金→傳說傭兵」，晉升給獎金。
- **AI 戰記**：`genBattleTale` 用 Pollinations 免費生成戰記，超時用模板。

### 4.4 城鎮建築（Index.html `BLDG` + `LSPOTS`）
公會(委託)/人員隊伍/背包倉庫/酒館(招募)/武具店/旅館(回血)/黑市(稀有交易)/領主城堡(主線·領船)/冒險者名冊。
- **城郊地區地圖** `regionMapHTML()`：陸地邊界感（森林/山脈/海岸線/大道），地標可點進建築。
- 海洋相關建築（貿易所/出航/船塢/船商/艦隊）依解鎖狀態顯示。

## 5. 海洋系統（海）

> **解鎖節奏**：新玩家**不自帶船**。地城探到**第 5 層** → 到領主城堡**領新手船艦**（`apiShip_ op:'claim'`）→ 開放貿易/海戰。
> 探到**第 10 層** → 解鎖船商/艦隊（甩手掌櫃玩法）。前端 `seaOpen()` / `fleetOpen()` 控制顯示。

### 5.1 船艦（Data.gs）
- 起始船 `startShip()`：初心號，hull60/cannon6/crew8/speed6/gunTier1/tier1。
- **船級 `FLAG_TIERS`**（小帆船→快帆船→巡防艦→主力戰艦）：每級決定 hull/cannon/cargo/crew/gunTier 的**上限**。升級到上限要「船體擴建」(`classup`) 提升船級。
- **砲管等級 gunTier**（`flagGunMul`：每級 +15% 砲擊，上限受船級 maxGun 限制）。
- **砲彈類別 `AMMO`**（開戰前裝填）：實心彈(對船身高)/鏈彈(削敵還擊·助逃)/霰彈(強化接舷俘虜)。
- **船塢升級 `SHIP_UP`**：強化船身/加裝火砲/擴充貨艙/招募水手/改良帆裝（費用隨次數指數上升，受船級上限）。

### 5.2 貿易（Code.gs `apiTrade_` + Data.gs）
- **10 座港市 `MARKETS`**，各有 `cheap`(產地便宜)/`dear`(需求貴)/**`sig`(招牌名產)**。
- **14 種貨品 `GOODS`**（鹽…龍涎香）。價格 `tradePrice = base × 產地/需求modifier × 隨機波動 × 世界事件`。
- **名產（sig）**：每港一項招牌貨，產地最便宜、運到別處大賺，讓航線有辨識度。地圖與 portInfo 會標 🏆。
- 買賣練**商業**等級；出航練**航海術**。

### 5.3 世界隨機事件（Data.gs `worldEvent`）
每日一件、全世界共享（以當日 seed 決定）：風平浪靜/搶購熱潮(某貨↑)/行情崩盤(某貨↓)/港口慶典(某港↑)/海盜猖獗(掠奪更豐)。影響 `eventPriceMul`。

### 5.4 航海（`apiVoyage_`）
- 港間距離 → 天數（航海術縮短）。途中 1~3 個海上事件 `rollSeaEvent_`：順風/暴風雨(掉貨)/漂流物(撿貨)/友善商船/海盜勒索(交涉可免)/海圖殘片。
- 有投資 → 每趟領商會分紅。

### 5.5 海戰／掠奪
- **戰術 stance**：`sink`砲擊擊沉(打撈半數)/`board`接舷俘虜(全額+奪船)/`kite`打帶跑(邊打邊逃，速度差越大越好逃)。
- **PvE**：`apiNaval_`(隨機敵船)、`apiSea_`(海域 NPC 商船，每日刷新、防重複掠奪)。敵船池 `ENEMY_SHIPS`。
- **PvP**：`apiRaidPlayer_` 掠奪其他玩家（新手 deepest<3 受保護）。目前**擊沉搶2成/接舷搶3成金幣＋搶貨**，
  ⚠️ **尚未實作「奪玩家船＋贖金」系統**（見 §8 待辦）。
- **偵查 `apiScout_`**：先偵查得勝率估計＋建議戰術，成功則下次攻擊有**奇襲先手齊射**；沒察覺/隱匿專長有機率暴露。
- **鹵獲船 `apiPrize_`**：接舷俘虜 NPC 船 → 編入艦隊或拆解換金。
- **說服船員入夥 `tryPersuade_` / `apiCaptive_`**：接舷俘虜（board 勝）NPC 時，除了奪船，還有機會**招降敗方一名船員**（交涉專長 +35%、🍇霰彈 +10%、總管/副手情義小加成，上限 85%）。玩家決定**收編入團**（加進 roster）或**放走換贖金**（金幣）。**招到的職業依敵船所屬勢力、稀有度依敵船強度階**（越強的船招到越稀有）。隊伍滿則只能換贖金。僅限 PvE NPC。
- **俘虜船長 `apiCaptain_`**：board 勝必俘虜**敵船船長**（屬某勢力）。玩家決定：**贖回原勢力**（金幣＋緩和該勢力仇恨 +50）／**賣給敵對勢力**（金幣 ×1.6，但徹底得罪原勢力 -80）／**放走**（無金幣，但大加好感 +120）。

### 5.9 NPC 勢力 / 好感度（Data.gs + Code.gs）

## 9. 開發・測試・部署慣例

- **分支**：功能開發在指定 feature 分支（如 `claude/gas-...`），commit 訊息用繁中、清楚描述。**push 到 `main` 才會觸發部署**。
- **部署**：push `main` → GitHub Action 跑 clasp push + deploy（更新既有部署 `DEPLOY_ID`，`/exec` 網址不變）。
- **測試**：用 Playwright（chromium 在 `/opt/pw-browsers/`，`NODE_PATH=/opt/node22/lib/node_modules`）。
  本機用 node http server 服務 `Index.html`，`page.evaluate` 直接 seed `window.P` 再呼叫 `renderXxx`/`openXxx` 驗證渲染與互動。離線 mock 有 250ms 假延遲，測 claim 等流程要留等待時間。
- **改動守則**：
  1. 動到伺服器規則/資料 → **同步 Index.html 的 `*C` 鏡像**。
  2. 動到 player 欄位 → 確認 `ensureShape_`（Store.gs）與 `ensureC`（Index.html mock）都補洞。
  3. 頭像相關 → 別碰 `pthumb` 的「回傳原網址」原則。
  4. 新增港口 → 同步 `MARKETS`/`MARKETS_C` + `GARRISONS`/`GARRISONS_C`。
